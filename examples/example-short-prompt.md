# Example Short Prompt

Rough idea: I want to build a notification system for overdue tasks.

Right now, users often miss deadlines because there is no clear reminder flow. I want a first version where the system detects overdue tasks, sends reminders to the right people, and avoids spamming users with duplicate notifications.

Help me turn this into a clear spec, including the problem statement, goals, user flow, constraints, edge cases, open decisions, and success criteria. Then generate a scoped implementation prompt for the first build slice.
